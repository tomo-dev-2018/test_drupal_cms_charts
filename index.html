Common Issues & Solutions

Charts not rendering / JavaScript errors

The charting library files are not in the correct location.

Solution:

  1. Check that ApexCharts files are in /web/libraries/apexcharts/
  2. Clear Drupal cache: drush cr
  3. Check browser console for specific error messages

Composer doesn't download the JavaScript library

Your composer.json is not properly configured for front-end assets.

Solution:

Ensure your repositories section is an array (not an object) and includes asset-packagist:

"repositories": [
    {
        "type": "composer",
        "url": "https://packages.drupal.org/8"
    },
    {
        "type": "composer",
        "url": "https://asset-packagist.org"
    }
]

Views aggregation not working correctly

Charts require aggregated data to display properly.

Solution:

  • Enable "Use aggregation" in Views Advanced settings
  • Set appropriate aggregation type (Count, Sum, etc.) for data fields
  • Ensure you have both a label field and a data field configured

Best Practices

📈 Data Management

  • • Keep CSV data simple - first row as headers, clean values
  • • Use Views for dynamic data that changes frequently
  • • Cache Views-based charts appropriately for performance

🎨 Visual Design

  • • Choose chart types that match your data story
  • • Keep color schemes consistent across your site
  • • Consider mobile responsiveness when placing charts

⚡ Performance

  • • Limit the number of data points for better performance
  • • Use aggregation in Views to reduce data processing
  • • Consider lazy loading for below-the-fold charts

🔧 Maintenance

  • • Document your Views configurations for team members
  • • Export chart configurations with your site config
  • • Test charts after Drupal and module updates
